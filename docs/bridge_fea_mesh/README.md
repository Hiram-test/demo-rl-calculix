# 桥梁构件有限元与网格需求语料库

## 目的

本目录把项目主问题从“寻找一种通用网格优化算法”改为：

> **先识别工程师要用有限元回答什么问题，再选择相应的模型层级、误差量、网格规则和验证证据。**

资料分为两套互补语料：

1. **案例与规则语料**：100 个可追溯案例或问题信号，用于理解正式方法、构件类型和研究边界；
2. **工程师直接问题语料**：81 个公开帖子/FAQ 拆出的 174 个原子问题，用于理解工程师实际卡在哪里、想获得什么形式的帮助。

两套语料都属于可扩展种子集，不宣称已经字面穷尽全球桥梁有限元案例，也不构成统计抽样调查。

## 直接问题语料的当前信号

在 174 个原子问题样本中：

- 84 个发生在问题定义与建模阶段；
- 37 个发生在验证、收敛与验收阶段；
- 25 个是直接网格问题；
- 11 个是结果提取与解释；
- 9 个是自动化、复用与规模化；
- 8 个是求解稳定性和数值执行。

这说明公开提问中的主要障碍往往不是“单元尺寸设多少”，而是模型层级、连接传力、边界、荷载阶段、接触、结果语义和可信度证据。该频次只描述当前软件社区样本，不能外推为全球工程师比例。

## 关键结论

1. **工程界不存在脱离用途的唯一“最佳网格”。** 同一模型对位移、局部疲劳应力、接触压力、裂缝路径、屈曲波形和冲击失效的网格要求不同。
2. **模型理想化通常先于网格密度。** 梁/壳/实体、系统模型/局部模型、专用支座单元/实体接触的选择错误时，继续细化不能修复问题。
3. **连接与传力是高频障碍。** MPC、tie、接触、嵌入、非匹配网格和梁壳实体耦合经常先于网格尺寸决定结果是否有意义。
4. **最大应力不等于应加密区域。** 必须区分有限应力集中、数学奇异性、应力梯度、结构应力和目标导向误差。
5. **非线性软化必须处理网格客观性。** 混凝土开裂、损伤、局部化和侵蚀模型不能靠单纯减小单元尺寸获得唯一答案。
6. **接触、动力和施工阶段是联合离散问题。** 接触面密度、时间步、移动荷载路径、事件点和施工阶段必须一起设计。
7. **工程师需要诊断和证据，而不只需要参数。** 自动系统必须解释异常来源、给出验证计划并说明停止条件。
8. **AI 的最佳入口是意图编译，而不是替代有限元。** AI 应从自然语言、规范和历史案例中生成可审计的分析与网格任务；最终结论由确定性网格、真实求解和收敛/验证检查裁决。

## 目录使用原则

每个新任务先填写 `bridge_mesh_intent.schema.json`。任何自动网格建议至少要回答：

- 当前工程决策是什么；
- 关注的结果量和区域是什么；
- 哪些峰值被视为奇异或不用于验收；
- 使用何种模型层级和单元类型；
- 构件、支座、钢筋和界面是否正确连通与传力；
- 荷载、预应力和施工阶段是否正确表达；
- 需要哪一种收敛或验证证据；
- 精度、时间和资源预算是什么；
- 哪些来源支撑这项选择；
- 哪些判断仍需工程师确认。

旧的 VM 数值结果文档已从本研究分支删除。Git 历史和既有 Actions 工件是不可变审计记录，不作为本目录当前结论。
