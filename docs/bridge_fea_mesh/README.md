# 桥梁构件有限元与网格需求语料库

## 目的

本目录把项目主问题从“寻找一种通用网格优化算法”改为：

> **先识别工程师要用有限元回答什么问题，再选择相应的模型层级、网格规则、结果协议和验证证据。**

资料分为三套互补语料：

1. **案例与规则语料**：100 个正式案例或问题信号，用于理解有效方法、构件类型和研究边界；
2. **工程师直接问题调研**：81 个公开帖子/FAQ 拆出的 174 个原子问题汇总，用于观察工程师在完整有限元流程中卡在哪里；
3. **桥梁网格专项问题语料**：36 个唯一来源页面拆出的 103 个可逐行审查原子网格问题，只保留尺寸、布局、过渡、连通、局部细化、子模型、奇异性、收敛、结果恢复和重划分问题。

三套语料都属于可扩展种子集，不构成概率抽样调查，也不代表全球桥梁工程师的真实频率分布。

## 从专项问题得到的核心要求

1. **“网格足够”必须绑定工程 QoI。** 位移、局部应力、接触压力、疲劳结构应力、板弯矩和支承合力需要不同的比较位置、提取协议和停止条件。
2. **网格布局首先服从桥梁构造事件点。** 主梁线、横梁、横撑、肋、支座、荷载点、截面变化、损伤边界和施工阶段边界必须稳定落入网格拓扑。
3. **局部尺寸必须和过渡、连接共同定义。** 最小尺寸本身不能回答共同边节点、相邻尺寸比、过渡层数、接口方法和结果关注区距离。
4. **必须阻止追逐不可收敛峰值。** 裂尖、点支承、尖角和某些焊接细节需要奇异性识别及固定位置、路径、条带或断裂参数等结果协议。
5. **网格比较必须固定后处理语义。** 节点平均、积分点、路径外推、结果梁积分和设计条带改变时，不能把结果差异全部归因于网格。
6. **全局—局部切换是网格系统的核心能力。** 系统要决定局部边界距离、传递量、重叠区一致性以及如何避免边界峰值和重复刚度。
7. **网格失败需要可操作诊断。** 输出应包含失败几何实体、短边或压印等机制、最小修复动作及其对物理模型的影响。
8. **重划分必须可重复。** 命名集、荷载、接触、路径、支座和结果取点在网格变化后不能悄然漂移。

## 目录使用原则

每个新任务先填写 `bridge_mesh_intent.schema.json`。任何自动网格建议至少要回答：

- 当前工程决策和目标结果量是什么；
- 结果在哪里、以什么协议提取；
- 哪些构造点、线、面必须与网格对齐；
- 哪些峰值属于奇异性或不得用于停止判断；
- 采用哪种单元类型、阶次和全局—局部层级；
- 局部尺寸、过渡和接口如何共同定义；
- 使用怎样的网格序列和容许变化证明充分性；
- 网格变化后哪些作用域和结果位置必须保持不变；
- 计算时间、单元数和人工干预预算是多少；
- 哪些技术规则有权威来源，哪些判断仍需负责工程师确认。

旧的 VM 数值结果文档已从本研究分支删除。Git 历史和既有 Actions 工件是不可变审计记录，不作为本目录当前结论。
