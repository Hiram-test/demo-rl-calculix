## 工程师在网格层面真正需要完成的工作

### 1. 知道“足够”是针对哪个结果量

反复出现的问题不是简单的“尺寸设多少”，而是：

- 位移已经稳定、应力仍变化，能否停止？
- 应比较全局最大值，还是每张网格中的同一物理位置？
- 接触压力、板弯矩、疲劳结构应力和支承反力是否需要不同的停止条件？
- 相邻两级网格变化多少才足以支持当前工程决策？

网格助手必须要求用户先声明：

`QoI + 位置 + 提取协议 + 容许变化 + 网格序列`

不得只以单元数量或云图平滑程度判断充分性。

### 2. 让细网格真正连接到粗网格

桥梁问题中大量困难来自：

- 局部细化产生悬空节点；
- 相邻桥面板网格尺寸差异过大；
- 共同边分段不一致；
- 粗细过渡进入最终关注区；
- 梁、壳、实体之间没有兼容节点或重复刚度。

工程师需要的不是一个单独的最小尺寸，而是完整的过渡合同：

- 共同边节点规则；
- 是否共形；
- 非共形连接方法；
- 相邻尺寸比；
- 过渡层数和距离；
- 过渡区能否用于结果验收。

### 3. 让网格服从桥梁构造事件点

桥面、正交异性板和斜交桥问题反复要求节点准确落在：

- 主梁线；
- 横梁和横撑；
- 加劲肋；
- 支座；
- 集中荷载位置；
- 截面变化；
- 损伤或缺梁边界；
- 施工阶段激活边界。

这说明桥梁网格首先是**构造拓扑和事件点问题**，其次才是连续尺寸场问题。参数化几何变化后，这些事件点还必须保持稳定。

### 4. 区分“网格不够”与“结果本来不收敛”

裂纹、点支承、尖角和焊接细节中，工程师经常遇到：

- 网格越细峰值越高；
- 最大值位置随网格跳动；
- 节点平均值和积分点值差异很大；
- 局部细化后位移收敛但峰值应力不收敛。

网格助手必须先判断结果量是否可收敛，再决定是否细化。必要时应改为：

- 固定物理位置的应力；
- 路径或条带平均；
- 结构热点应力；
- 距裂尖固定距离的量；
- `J` 积分或应力强度因子；
- 有限支承面积上的合力或平均量。

### 5. 保持网格与结果恢复协议一致

很多所谓网格问题其实发生在网格之后：

- 壳应力取节点平均还是积分点；
- 面内力如何积分为结果梁；
- 桩帽弯矩按墙厚条带还是全宽；
- 壳模型怎样恢复设计梁内力；
- 不同软件显示的“10×10网格”是否真是同一内部分析网格。

因此每次网格比较必须固定结果恢复协议。否则网格变化和后处理变化无法分离。

### 6. 决定何时从全桥网格切换到局部网格

工程师反复询问：

- 局部模型边界应离孔洞、裂纹或焊接细节多远；
- 应传递位移、力还是应力；
- 全桥梁模型何时切换成壳或实体；
- 局部细化后怎样检查与全局模型的一致性；
- 怎样避免边界峰值和重复刚度。

这意味着系统必须支持“全局—局部网格计划”，而不只是全模型单次划分。

### 7. 从复杂几何稳定生成网格

桥梁几何中的高频失败包括：

- 抑制构件后残留压印；
- 极短边和近零长度单元；
- 曲线和非平行边导致畸变板单元；
- 参数变化后网格控制失效；
- 重复节点、翻转单元和未连接边；
- 为了一个小构件迫使大范围实体改用四面体。

工程师需要系统指出：

`失败几何实体 + 失败机制 + 最小修复动作 + 修复对物理模型的影响`

而不是只返回“mesh failed”。

### 8. 网格变化必须可重复

网格收敛和批量任务中，工程师需要：

- 变化网格后仍保留接触、路径、命名集和边界作用域；
- 每级网格在同一物理位置提取结果；
- 参数化桥梁中支座、横撑和荷载事件点不漂移；
- 保存尺寸场、单元类型、阶次、质量和软件版本；
- 失败后可以重现和恢复。

所以网格自动化的评价指标不只是减少单元数，还包括复现率、作用域保持率和人工返工时间。

## 对 MeshPilot 的直接要求

仅从网格专项问题出发，系统至少需要八个模块：

1. **网格任务编译器**：将 QoI、位置、误差和资源预算写成结构化合同；
2. **事件点与构造线提取器**：把支座、横梁、肋、荷载和截面变化转成必须对齐的几何约束；
3. **网格可生成性诊断器**：识别小边、压印、重复实体、不可扫掠区和短单元；
4. **局部尺寸与过渡规划器**：同时给出细化域、尺寸比、过渡距离和接口方式；
5. **混合维度连接检查器**：检查梁—壳—实体的节点、偏置和刚度重复；
6. **收敛实验管理器**：生成受控网格序列，在相同物理位置提取相同 QoI；
7. **奇异性与结果协议检查器**：阻止系统追逐不可收敛峰值；
8. **可重复重划分管理器**：保持命名集、荷载、接触、路径和版本可追溯。
