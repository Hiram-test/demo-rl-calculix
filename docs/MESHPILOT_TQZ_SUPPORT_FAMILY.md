# MeshPilot TQZ(XII) 支座局部构件族

## 为什么先从这里开始

上传图纸为 `通桥[2023]2322A-II`，对应时速 350 km 高速铁路、跨度 31.5 m、梁高 3.0 m 的双线预制无砟轨道后张法预应力混凝土简支箱梁。

第一阶段不直接建立整孔预应力箱梁。整梁会同时引入预应力、钢筋、轨道荷载、支座边界和大量局部构造，难以区分 PSO 缺陷、建模缺陷和物理假设缺陷。

图纸第 34、36 个 PDF 页面（图纸页号 33、35）给出了两组天然的参数化支座尺寸表：

- TQZ(XII)-4500：Ag = 0.10g、0.15g、0.20g；
- TQZ(XII)-5000：Ag = 0.10g、0.15g、0.20g。

它们使用相同构件语义和求解流程，仅承载等级、地震参数及 A/B/C/D/H 尺寸发生变化，符合 Batch-Transfer PSO 的业务前提。

## 第一轮有限元边界

第一轮是算法验证模型，不是工程验算模型。

计划建立一个局部三维线弹性实体：

1. 支座足迹、垫石及周边局部混凝土；
2. 顶面通过支座足迹传递名义竖向合力；
3. 远端切面采用固定或对称约束；
4. 使用 C50 线弹性参数；
5. 在加载边缘和垫石交界处设置 6 个确定性网格控制 patch；
6. 以加载面平均竖向位移、弹性柔度和单元数作为第一轮指标。

第一轮明确不包含：

- 支座接触和摩擦；
- 混凝土开裂、塑性和损伤；
- 显式钢筋与预应力筋；
- 31.5 m 整梁的全局效应；
- 工程安全验收结论。

## “少砍几刀”实验设计

同一批六个构件按参数距离进行最近邻排序。每个算例同时运行：

- 冷启动、可行优先、无重复补位离散 PSO；
- 带 warm probe 门卫的 Batch-Transfer PSO。

对照必须报告：

- 每个算例新增的真实唯一 CalculiX 求解次数；
- 最终解是否满足单元预算；
- 最佳位移误差、柔度和单元数；
- warm start 来源及参数距离；
- 门卫接受或恢复完整预算；
- 重复离散位置补位次数；
- 总墙钟时间。

第一轮通过条件：

1. 六个 reference/coarse/优化算例全部完成；
2. 冷启动和迁移结果均为可行；
3. 每个优化均遵守真实 FE 硬上限；
4. 迁移总 FE 次数低于冷启动；
5. 平均解质量不出现不可解释的明显退化；
6. 至少保留一个故意打乱顺序或热点改变的门卫测试。
