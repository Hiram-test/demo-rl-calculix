# MeshPilot：真实热点候选区、阶梯式离散 PSO 与批量迁移

## 这次不再使用固定 3×2 区域作为主方法

旧 TQZ 基线把支座加载足迹机械划成 3×2 六块。它保留为对照组，但不再代表主方法。

新的候选区来自每个构件自己的真实粗网格计算：

1. 使用真实 Gmsh 生成均匀粗网格；
2. 使用真实 CalculiX 完成三维线弹性求解；
3. 根据 CalculiX 节点位移重构 C3D4 单元等效应力；
4. 将局部块体上部划为 5×4×3 个虚拟 cell；
5. cell 分数由应力水平和与相邻 cell 的应力差共同构成；
6. 相邻高分 cell 合并；
7. 取前 6 个区域作为本次 PSO 的固定候选区。

候选区一经粗网格确定，在该算例的一次 PSO 搜索中不移动。PSO 只决定六个热点区域分别使用 L0、L1、L2 或 L3。

## 算法层改进：阶梯式离散 PSO

普通基线仍采用“连续位置更新后四舍五入”。新算法直接在离散等级上运动：

- 每一维每一轮最多上升一级、下降一级或不动；
- 个体最优和全局最优给出方向；
- 二者同向时提高移动概率；
- 二者冲突时降低移动概率；
- 保留少量随机一步探索；
- 所有重复位置仍在求解前补到最近未评价位置。

当群体连续两轮没有改善时，算法不会立即随机重启或停止，而是先真实计算当前全局最优的所有未评价 ±1 邻居。六维问题最多只有 12 个一步邻居。邻域全部检查后仍无改善，才确认本地结束。

## 业务层改进：热点对应关系迁移

迁移不再复制“旧 patch 1 → 新 patch 1”。每个热点保存：

- 相对块体的归一化中心；
- 归一化尺寸；
- 粗网格热点分数；
- 最优网格等级。

新构件完成自己的粗网格热点检测后，通过中心、尺寸和分数进行一对一匹配。匹配成功的热点接收历史等级；未匹配热点保持 L0。随后必须执行一次真实 CalculiX warm probe：

- probe 可行且目标值不比当前 coarse 基线差超过 20%，才缩减 FE 预算；
- 否则拒绝迁移并恢复完整 32 次唯一 FE 预算。

## 四组真实对照

| 组别 | 候选区域 | PSO | 批量迁移 | 目的 |
|---|---|---|---|---|
| A | 固定加载足迹 3×2 | 连续位置取整 PSO | 无 | 原始基线 |
| B | 真实 coarse 应力热点 | 连续位置取整 PSO | 无 | 单独验证热点选择 |
| C | 与 B 相同热点 | 阶梯式离散 PSO＋一步邻域收尾 | 无 | 单独验证 PSO 算法 |
| D | 与 B 相同热点 | 与 C 相同 | 热点匹配＋真实门卫 | 验证批量业务收益 |

每组、每个算例的目标函数都来自真实 Gmsh/CalculiX；不使用代理模型、经验公式替代、假求解器或降低维度的模拟结果。每个优化运行最多新增 32 次唯一 CalculiX 求解。

## 必须报告的结果

- 六个构件是否全部完成；
- 四种方法是否全部满足单元预算；
- 每种方法的真实唯一 FE 次数；
- 最佳相对位移误差、目标值和单元数；
- 每个构件的六个粗网格热点位置和分数；
- 阶梯 PSO 的一步邻域求解次数；
- 热点迁移匹配关系、来源距离和门卫结果；
- A→B、B→C、C→D 三段收益分别是多少。

## 解释边界

模型仍是支座附近局部 C50 三维线弹性块体，用于研究网格优化算法。接触、摩擦、钢筋、预应力、混凝土开裂和整孔箱梁全局响应未包含，所有结果仍需工程人员复核，不能作为支座或梁体的验收结论。
