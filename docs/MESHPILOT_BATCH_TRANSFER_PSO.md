# MeshPilot：受约束批量与“少砍几刀”的 Batch-Transfer PSO

## 论文只聚焦两个问题

### 1. 相邻工程用户的批量与受控外推

目标用户不是完全没有工程背景的公众，而是了解构件、材料、载荷和设计意图，但不擅长网格优化和 PSO 调参的结构设计、桥梁细节、BIM/CAD、制造工艺、检测养护或初级分析人员。

用户可以用自然语言描述任务；大模型只负责生成受约束的 JSON 批量合同。确定性执行层负责：

1. 展开参数扫掠；
2. 判断每个算例属于模板内插、受控外推还是不支持；
3. 模板内算例自动运行；
4. 外推算例标记 `review_required=true`；
5. 超出硬边界的算例拒绝运行；
6. 输出每个算例的模型卡、网格方案、PSO 记录和审查状态。

这里的“普通用户外推”不是让用户随意建立任何有限元模型，而是在专家预先验证的模板族上进行新的参数组合和批量方案比较。

### 2. Batch-Transfer PSO

热点粗筛把搜索缩减到少量 patch。粒子是离散网格等级：

\[
\mathbf l=[l_1,\ldots,l_m],\qquad l_i\in\{L0,L1,L2,L3\}.
\]

本工作不发明复杂的新速度公式。核心问题只有一个：

> 在昂贵的有限元黑箱评价中，怎样少做没有必要的 CalculiX 调用？

## “木棍砍树”式的六条规则

### 规则一：已知的粗网格不再求第二次

每个算例在热点筛选前已经运行一次 uniform coarse 网格。PSO 的全 L0 粒子就是同一个网格，因此直接把预处理结果放入 PSO 缓存，不再启动第二次 CalculiX。

这不是近似，也不改变搜索结果，只删除一次确定性的重复求解。

### 规则二：重复粒子要补位

连续 PSO 位置四舍五入后，不同粒子可能落到同一个离散等级向量。缓存虽然能阻止重复 FE，但原来的粒子名额仍被浪费。

现在若某位置已经评价，系统就在 L0-L3 整数格上寻找离原连续位置最近的未评价位置。只要搜索空间还没有耗尽，每个粒子名额都会带来一个新网格方案。

距离使用最朴素的等级距离；实现采用从当前整数位置向相邻等级逐步展开的 best-first 搜索。

### 规则三：先看是否过线，再比较分数

个人最优和全局最优采用可行优先排序：

1. 可行网格一定优于不可行网格；
2. 都可行时比较目标函数；
3. 都不可行时先比较超预算程度，再比较目标函数和误差。

因此，只要搜索过程中找到过一个满足单元预算的方案，最终输出不会被一个目标值看似更小、但已经超预算的方案替换。

### 规则四：L0 和 warm start 同时保留

迁移模式下：

- 粒子 0 保留安全的全 L0 基线；
- 粒子 1 使用历史 warm start；
- 一部分粒子使用 warm start 的小扰动；
- 剩余粒子保持随机。

旧实现会让 warm start 覆盖 L0 槽位；现在两者同时保留。

### 规则五：直接限制真实 FE 次数

原方法通过缩小粒子数和迭代数间接控制成本。现在配置直接提供：

```json
"max_unique_evaluations": 32
```

相似历史算例存在时，使用：

\[
B(d)=B_{\max}-\exp(-d/s)(B_{\max}-B_{\min})
\]

其中 `d` 是归一化参数距离，`B` 是允许的真实唯一 FE 调用数。PSO 一旦达到预算立即停止，不再用“粒子数 × 代数”猜测实际成本。

### 规则六：旧答案先试一次，再决定是否省预算

找到最近历史算例后，先把历史等级场投影到当前热点，并真实求解一次：

- warm 方案不劣于当前 coarse 基线：允许按相似度缩减 FE 预算；
- warm 方案不可行或明显更差：保留 warm 粒子，但恢复完整冷启动预算；
- warm 与 coarse 相同：直接复用 coarse 结果，不再求解。

这一规则用于阻止“参数很近，但热点突然移动”造成的负迁移。

## 批量顺序也是算法的一部分

若算例按 Excel 原始顺序运行，当前算例只能从已经完成的历史中迁移。相同的一批构件，仅改变运行顺序就可能改变总 FE 次数。

`batch_order="nearest_path"` 使用简单贪心规则：

1. 从参数描述向量的中间算例开始；
2. 每次选择距离已完成集合最近的未完成算例；
3. 不支持算例放在最后并拒绝运行。

它不求解复杂的旅行商问题，只确保相邻执行任务尽量相似，与“从最近已完成算例迁移”的业务逻辑一致。若需要严格保持用户输入顺序，可设置：

```json
"batch_order": "input"
```

## 请求合同中的新增字段

```json
{
  "batch_order": "nearest_path",
  "transfer_min_evaluations": 8,
  "transfer_guard_ratio": 1.05,
  "pso": {
    "max_unique_evaluations": 32,
    "refill_repeats": true
  }
}
```

含义：

- `batch_order`：输入顺序或最近邻批次顺序；
- `transfer_min_evaluations`：高相似度迁移仍至少保留的真实 FE 预算；
- `transfer_guard_ratio`：warm probe 相对 coarse 允许的轻微目标波动；
- `max_unique_evaluations`：冷启动单算例的真实 FE 硬上限；
- `refill_repeats`：离散位置重复时是否自动补到最近未评价位置。

## 评价指标

### 批量业务

- 批量合同成功展开率；
- 模板内、外推和拒绝判定准确性；
- 专家复核触发率；
- 批次执行顺序与相邻参数距离；
- 单算例与整批周转时间；
- 不合规外推被正确阻止或升级的比例。

### PSO

- 达到相同解质量所需的真实 FE 次数；
- 每轮重复位置数和自动补位数；
- warm guard 接受/回退次数；
- 最终误差、单元数和可行率；
- 热启动来源距离与节省 FE 次数的关系；
- 固定 `max_unique_evaluations` 下的最好可行方案。

## 批量迁移成立的边界

批量与迁移只适用于：

1. 同一构件或同一模板族；
2. 相同物理模型和求解流程；
3. 变化集中在有限几个参数；
4. 任务重复出现；
5. 热点区域和合理网格分配存在可迁移关系。

若拓扑、材料模型、接触关系或求解类型发生根本变化，不应强行迁移。

当前 VM 仍使用仓库已有的小型二维 Gmsh/CalculiX 模板，只验证机制与真实 FE 调用。三维桥梁构件泛化和目标用户可用性，需要后续参数化构件批次与用户实验。
