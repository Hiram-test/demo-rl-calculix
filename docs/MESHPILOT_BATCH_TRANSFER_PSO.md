# MeshPilot：普通工程用户的批量外推与 Batch-Transfer PSO

## 论文只聚焦两个问题

### 1. 相邻工程用户的批量与受控外推

目标用户不是完全无工程背景的公众，而是了解构件、材料、载荷和设计意图，但不擅长网格优化和 PSO 调参的结构设计、桥梁细节、BIM/CAD、制造工艺、检测养护或初级分析人员。

用户通过自然语言描述任务；大模型只负责生成一个受约束的批量任务合同。确定性执行层负责：

1. 展开参数扫掠；
2. 判断每个算例属于模板内插、受控外推还是不支持；
3. 模板内算例自动运行；
4. 外推算例标记 `review_required=true`；
5. 超出硬边界的算例拒绝运行；
6. 输出每个算例的模型卡、网格方案、PSO 记录和审查状态。

这里的“普通用户外推”不是让用户随意建立任何有限元模型，而是在专家预先验证的模板族上进行新的参数组合和批量方案比较。

## 一般什么时候真正需要批量分析

批量并不只是“算例数量多”，而是同一套分析流程反复用于一组相似任务。典型业务场景如下。

### 参数化设计比较

同一种桥梁构件需要比较不同板厚、孔径、圆角、加劲肋尺寸、材料等级或网格预算。例如 5 种孔径、4 种板厚和 3 种加劲肋方案会形成 60 个相似算例。建模逻辑不变，只有少量参数变化，因此适合自动展开和跨算例复用。

### 多工况验算

结构保持不变，但需要批量改变车辆位置、轮载大小、风向、温度梯度、支座沉降或施工阶段。传统流程中工程师不断复制模型、修改荷载并整理结果，容易漏工况或填错参数。

### 一批相似构件

同一项目或资产群中存在大量同类构件，例如不同位置的横隔板、相似焊接细节、多座同类型桥梁的局部构造，或一批具有不同腐蚀减薄程度的构件。专家建立一次可信模板，相关工程人员批量填入构件参数。

### 高频设计迭代

设计人员反复修改构造后，需要快速重新评价。智能体可把“将板厚改为 16、18、20 和 22 mm，并保持孔边至少 L2 加密”直接转换为四个受约束算例，而不需要 CAE 专家逐个复制和配置模型。

### 不确定性、制造容差和敏感性分析

板厚、焊缝尺寸、材料参数、腐蚀深度和荷载位置均可能存在离散。围绕标称值生成批量样本，可用于判断哪些参数变化会显著影响结果。

### 检测养护和周期性更新

模型模板不变，但每次检测会更新腐蚀、裂缝、变形或交通荷载数据。系统可以对多座桥梁或多个构件批量更新分析，并把异常结果送交专家复核。

## 批量机制成立的基本条件

批量与迁移只适用于同时满足以下条件的任务：

1. 属于同一类构件或同一模板族；
2. 使用相同的物理模型和求解流程；
3. 变化集中在有限几个参数；
4. 任务会重复出现，而不是仅计算一次；
5. 不同算例之间的热点区域或合理网格分配存在可迁移关系。

若模型拓扑、材料模型、接触关系和求解类型均发生根本变化，则不应把它们强行归为同一批次，也不应迁移前序 PSO 解。

## 2. Batch-Transfer PSO

热点只用于把候选区域缩减到少量 patch。粒子仍然是离散等级：

\[
\mathbf l=[l_1,\ldots,l_m],\qquad l_i\in\{0,1,2,3\}.
\]

标准冷启动 PSO 对每个算例都重新随机初始化。Batch-Transfer PSO 利用批量业务中相邻算例通常很相似这一事实：

1. 计算当前算例与已完成算例的归一化参数距离；
2. 选择最近算例的最佳完整等级场；
3. 将该等级场投影到当前热点候选集；
4. 用它和其小扰动初始化部分粒子；
5. 相似度越高，粒子数和迭代数越小；
6. 相似度不足或首个算例使用完整冷启动预算。

优化目标不宣称降低了抽象速度更新的复杂度，而是直接减少真实有限元调用：

\[
N_{FE}^{transfer}<N_{FE}^{cold}.
\]

改进的核心不是拼接新的速度公式，而是针对昂贵有限元黑箱评估，利用批次相似性、离散缓存、热启动和自适应搜索预算减少真实求解器调用。

## 当前代码

- `meshpilot_batch_agent.py`：任务合同、批量展开、适用域判定、热点粗筛、真实 CalculiX 评估和报告；
- `meshpilot_pso.py`：离散 PSO、缓存、跨算例热启动和相似度自适应预算；
- `examples/meshpilot_batch_request.json`：三个模板内算例和一个受控外推算例；
- `tests/test_meshpilot_*.py`：批量适用域、热点筛选、缓存和热启动回归测试；
- `.github/workflows/meshpilot-batch-pso-vm.yml`：真实 Gmsh/CalculiX 虚拟机对照测试；
- `docs/MESHPILOT_BATCH_TRANSFER_PSO_VM_RESULTS.md`：虚拟机实验设置、结果和解释边界。

## 评价指标

### 普通工程用户与批量逻辑

- 批量合同成功展开率；
- 模板内、外推和拒绝判定准确性；
- 每批任务人工输入字段数；
- 专家复核触发率；
- 单算例与批量总周转时间；
- 重复性任务中的人工配置时间；
- 不合规外推被正确阻止或升级的比例。

### PSO

- 达到相同目标值所需的真实 FE 次数；
- 总墙钟时间；
- 最终误差和单元数；
- 内插算例与外推算例的性能差异；
- 热启动来源距离与性能收益的关系；
- 与完整冷启动 PSO 相比的解质量保持率。

## 现阶段边界

当前 VM 使用仓库已有的小型二维 Gmsh/CalculiX 模板，只验证批量执行、受控外推和 PSO 迁移机制。它不能代替后续三维桥梁构件模板，也不能证明非专家可用性；后续仍需三维模板和目标用户实验。
