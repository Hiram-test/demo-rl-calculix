# MeshPilot：面向相邻专业工程人员的自然语言三维有限元局部网格优化智能体

> 状态：研究与产品定义备忘录  
> 日期：2026-07-22  
> 目标：明确“谁需要用、为什么需要、能用到什么程度”，并提出热点引导、离散加密等级与改进 PSO 相结合的技术路线。

---

## 0. 核心结论

本项目的目标用户不是没有工程背景的社会公众，也不是取代有限元专家，而是以下这类**相邻专业工程人员（bounded non-expert / adjacent engineer）**：

- 能理解结构、材料、载荷、约束和设计意图；
- 能判断输入是否符合真实工程场景；
- 但不以网格策略、求解器参数、误差估计和优化算法为日常专长；
- 经常需要完成标准化、重复性较强的局部分析与设计迭代；
- 最终结果仍处于专家定义的模板、边界和验证规则之内。

项目的产品逻辑不是“让任何人都能做有限元”，而是：

> **由专家建立经过验证的模型模板、求解规则和安全边界；由相邻专业工程人员用自然语言表达设计任务、重点区域、精度和预算；由智能体完成任务编译、工具编排、热点候选生成、PSO 网格分配、验证与可追溯报告；超出模板或风险边界时自动升级给专家。**

技术核心为：

\[
\boxed{
\text{工程意图}
\rightarrow
\text{结构化分析合同}
\rightarrow
\text{粗网格求解与候选区域}
\rightarrow
\text{热点引导的离散 PSO}
\rightarrow
\text{确定性验证}
\rightarrow
\text{用户交互修改与专家升级}
}
\]

## 1. 为什么不是“只有专家才需要模拟”

### 1.1 工业需求并不是让非专业人员创建任意模型

行业中的“simulation democratization”通常不是让用户自由定义任意弱形式、材料模型或非线性求解器，而是把**已经验证的高价值流程标准化、模板化并部署给更大的工程团队**。

NAFEMS 在 2023 年成立 Upfront Simulation Working Group，明确研究“运行快、只需较少专业知识”的几何仿真工具，目标是在产品设计阶段把仿真扩展到非专职仿真人员，同时强调必须研究这些方法的适用限制、工程流程位置和治理要求。[R1]

COMSOL 的 Application Builder 采用相同业务模式：仿真专家建立模型和专用界面，应用使用者只接触与任务有关的输入和结果，不需要理解通用求解软件的全部细节；官方列出的部署对象包括设计团队、制造部门、过程操作人员、试验实验室、客户和合作方。[R2][R3]

Dassault Systèmes 的 MODSIM 也把仿真民主化定义为：将高价值仿真流程标准化和模板化，使设计人员和非专家可以重复使用高质量最佳实践，同时释放仿真专家去处理更高价值工作。[R4]

因此，合理目标是：

- **专家生产和维护可信流程；**
- **相邻工程人员消费和调整可信流程；**
- **智能体负责把两者连接起来。**

### 1.2 “普通用户”应改称“相邻专业用户”

本项目不面向完全不了解工程的人。最合适的用户具有领域知识，但缺少深度 CAE 技能。

| 用户角色 | 已具备的知识 | 缺少或不愿承担的工作 | 典型需求 |
|---|---|---|---|
| 结构设计工程师 | 结构体系、设计规范、材料、载荷 | 三维网格控制、求解器调参、优化算法 | 设计变更后快速判断局部风险 |
| 桥梁细节/构造工程师 | 横隔板、加劲肋、焊缝、孔口等构造知识 | 建立并验证完整局部 FE 工作流 | 比较多个细节方案 |
| BIM/CAD 工程师 | 几何、构件编号、装配关系 | 将 CAD 转成可信分析模型 | 在模型修改时获得早期反馈 |
| 制造/工艺工程师 | 加工、焊接、开孔、装配约束 | 高级有限元和网格优化 | 评估工艺变更或可制造性影响 |
| 检测与养护工程师 | 病害位置、腐蚀、减薄、加固方案 | 从现场信息构建局部分析模型 | 做维修方案筛选和优先级判断 |
| 初级分析工程师 | 有限元基础、材料与边界条件 | 高级网格策略和自动优化 | 完成大量标准化算例并交专家复核 |
| 技术负责人/项目经理 | 需求、风险、进度和成本 | 直接操作有限元软件 | 查看可追溯结果和方案比较，不直接建模 |

### 1.3 明确排除的用户和任务

以下情形不应由系统自主完成：

- 用户不能给出或确认材料、单位、约束和载荷；
- 新型材料本构、断裂、复杂接触、流固耦合等超出验证模板；
- 几何和工况明显超出模板参数范围；
- 最终法规验算、认证、桥梁安全签字和事故责任鉴定；
- 结果对生命安全有直接影响但缺少专家复核；
- 网格收敛、平衡检查或求解稳定性没有通过。

系统必须能说“当前任务需要专家”，而不是为了完成对话自动猜测。

## 2. 这些用户是否真的有需求

### 2.1 需求一：早期设计变更发生得快，专家队列跟不上

Ansys 将传统仿真难以部署给设计团队的障碍总结为复杂度、易用性和速度，并把设计工程师以及希望卸载常规分析的分析师列为 upfront simulation 的目标用户。[R5]

Caterpillar 的公开案例给出了更直接的组织逻辑：随着产品开发加快和产品复杂度提高，仿真使用从 CAE 专家扩展到设计工程师，以便在概念阶段快速迭代；同时，企业把专家知识封装成可控的 Web 应用，解决传统电子表格在版本、知识产权和访问控制方面的问题。[R6]

对于桥梁或钢结构局部构件，类似迭代包括：

- 横隔板孔径、圆角和加劲肋位置变化；
- 板厚、肋厚、焊缝附近构造变化；
- 施工、运输或安装方案改变约束；
- 检测发现腐蚀、减薄或局部损伤后比较维修方案；
- 招投标或初步设计阶段快速筛除明显不合理方案。

这些任务通常不是一次性的原创研究，而是相似模板上的高频变型。

### 2.2 需求二：专家时间应集中在高风险和新问题上

Noesis SimApps 的业务描述强调，把重复分析任务和最佳实践封装后，设计与仿真部门之间的迭代等待可明显减少；非专家不需要了解底层仿真细节，而专家保留流程设计和质量控制职责。[R7]

系统的经济价值可抽象为：

\[
C_{\text{template}} + N C_{\text{agent-run}}
<
N C_{\text{expert-manual}},
\]

其中：

- \(C_{\text{template}}\)：专家首次开发、验证和维护模板的成本；
- \(C_{\text{agent-run}}\)：相邻用户运行一次标准流程的成本；
- \(C_{\text{expert-manual}}\)：专家手工完成一次完整分析的成本；
- \(N\)：类似任务的重复次数。

因此，本系统只在**任务重复度足够高**时具有业务价值。对于一次性的、全新且高风险的模型，直接交给专家更合理。

### 2.3 需求三：不仅要易用，还要统一、可追溯和可治理

NAFEMS Simulation Governance & Management Working Group 强调仿真的可信度、治理和最佳实践会直接影响盈利、工期、产品质量、性能和安全。[R8]

仅仅提供一个“更简单的 GUI”不能解决：

- 不同人员采用不同模型假设；
- 单位和材料版本不一致；
- 结果文件散落且无法追踪；
- 同一问题重复建模；
- 用户不知道结果是否超出适用范围；
- 专家无法还原一次分析为什么得到该结果。

智能体必须自动保留：

- 用户原始意图；
- 结构化分析合同；
- 模板与版本；
- 单位、材料、载荷和边界条件；
- 几何哈希和选择区域；
- 网格策略和 PSO 评估记录；
- 求解器版本和日志；
- 验证检查；
- 用户修改和人工批准记录。

### 2.4 需求四：组织内已有大量“影子工程工具”

NAFEMS 收录的 EASA 案例指出，工程组织中常存在大量依赖电子表格、脚本和模型的关键流程，但这些工具在版本控制、访问控制、知识产权和审计方面存在风险；低代码应用平台的价值是把专家模型转成受控、可复用的企业应用。[R9]

MeshPilot 的竞争对象不只是 Abaqus、CalculiX 或 Ansys 的 GUI，还包括：

- 由个人维护的 Python 脚本；
- Excel 经验公式；
- 复制粘贴的旧输入文件；
- 依赖某位专家记忆的网格流程；
- 设计部门和 CAE 部门之间的邮件往返。

## 3. 用户可以用到什么程度

### 3.1 三类任务风险

#### R1：早期筛选和方案比较——可由相邻用户自主使用

- 比较 3–10 个参数化设计变体；
- 找出高风险局部区域；
- 在固定预算下产生合理网格；
- 输出“建议继续/需要专家分析”的初筛结论；
- 不用于最终签字。

#### R2：详细设计辅助——相邻用户运行，专家复核

- 使用经过验证的三维局部模板；
- 用户修改有限范围内的几何、材料和载荷参数；
- 智能体执行热点与 PSO 网格优化；
- 专家审查模型卡、收敛信息和最终结果。

#### R3：高风险或超模板任务——必须升级专家

- 非线性、断裂、接触、疲劳寿命、复杂组合工况；
- 结果用于最终法规验算或安全签字；
- 输入超出模板训练/验证范围；
- 多次验证失败或不同模型结论冲突。

### 3.2 智能体自主程度 A1–A3

为避免与网格加密等级混淆，智能体自主级别使用 A，网格级别使用 L。

#### A1：任务编译助手

- 理解自然语言；
- 生成分析合同和模型卡；
- 找出缺失参数并询问；
- 不自动执行求解，用户确认后才运行。

#### A2：受控执行智能体

- 调用模板、网格、求解器、热点、PSO 和验证工具；
- 只在白名单动作内自动处理常见失败；
- 输出多个 Pareto 方案；
- 用户批准最终选择。

#### A3：受控闭环优化智能体

- 根据用户反馈继续优化；
- 使用上一轮结果热启动 PSO；
- 自动并行、缓存和选择高保真评估；
- 达到目标后停止；
- 仍不能越过 R3 风险门或替代专家签字。

第一篇论文建议实现 A2，并展示少量 A3 的交互能力。

## 4. 为什么大模型/智能体有价值，而不只是一个表单

现有工业仿真应用已经证明模板化部署有需求，因此大模型的贡献不能只写成“自然语言输入”。FeaGPT 已经展示了自然语言到几何、网格、求解和分析的端到端流程。[R10] 2026 年的一项受约束自然语言有限元研究进一步指出，为降低可靠性风险，应限制 LLM 只处理前端任务和结构化规范，确定性模板负责弱形式和求解器核心。[R11]

本项目中，大模型真正适合做的是：

### 4.1 领域意图映射

把用户语言中的：

- “孔边”；
- “第二根加劲肋端部”；
- “固定端附近”；
- “这里不要太密”；
- “单元数控制在八万以内”；
- “优先保证位移，局部应力作辅助”；

映射为 CAD entity、空间区域、优先级、约束和目标量。

### 4.2 缺失信息管理

智能体必须区分：

- 可以由模板默认的参数；
- 可以从项目数据库检索的参数；
- 必须由用户确认的关键参数；
- 超出系统能力、需要专家的参数。

例如材料牌号缺失、载荷方向不明确、单位体系冲突时，不能直接运行。

### 4.3 工具编排与异常恢复

智能体根据状态调用：

```text
模板选择 → 几何检查 → 初始网格 → 求解 → 热点候选
→ PSO → 高保真验证 → 报告 → 用户修改
```

常见异常只允许从白名单中选择：

- 调整局部最小网格尺寸；
- 修复几何小特征；
- 回退上一网格；
- 降低并行度；
- 重新运行失败粒子；
- 标记任务需要专家。

### 4.4 可解释的业务交互

智能体应解释：

- 哪些输入来自用户；
- 哪些来自模板；
- 哪些是默认假设；
- 为什么某些区域进入候选集；
- PSO 为什么选择当前 L1/L2/L3 分配；
- 哪个约束阻止了更细网格；
- 结果适用于什么决策，不适用于什么决策。

### 4.5 多轮修改和热启动

表单适合一次性填参，但工程工作经常是：

> “孔边太密，第二根肋端部更重要；总单元数不变，再给我两个方案。”

智能体可以把这句话转成：

- 降低区域 A 的最低级别；
- 提高区域 B 的优先级或设为强制 L2；
- 保持 \(N_{\max}\)；
- 从上一轮粒子、缓存和代理模型继续优化。

这一能力是智能体相对固定仿真 App 的主要增量价值。

## 5. 热点与 PSO 的职责边界

### 5.1 热点不是最终决策器

热点模块只负责把全模型缩小为少量候选区域。候选来源可以是：

1. 用户强制指定区域；
2. 几何规则区域，如孔边、圆角、焊缝、肋端；
3. 粗网格结果中的高应力、高梯度或误差指示区域；
4. 候选区域的一圈邻域。

最终候选数建议控制在：

\[
m=4\sim 8.
\]

热点模块不应宣称提供严格误差控制；若使用应力热点，论文应称“热点引导网格优化”，而非“严格后验误差自适应”。

### 5.2 离散加密等级 L0–L3

对每个候选区 \(i\)，定义：

\[
l_i\in\{0,1,2,3\},
\]

其中：

- L0：保持基础网格；
- L1：轻度加密；
- L2：中度加密；
- L3：强加密。

一个粒子表示：

\[
\mathbf l=[l_1,l_2,\ldots,l_m].
\]

用户可施加：

- 强制区域：\(l_i\ge 1\) 或 \(l_i\ge 2\)；
- 禁止过密区域：\(l_i\le 1\)；
- 锁定区域：\(l_i=L_k\)；
- 总单元预算：\(N(\mathbf l)\le N_{\max}\)。

### 5.3 两种主要优化模式

#### 预算模式

\[
\min_{\mathbf l}\ E(\mathbf l)
\quad
\text{s.t.}\quad
N(\mathbf l)\le N_{\max}.
\]

用户语言：

> “总单元不超过八万，尽量提高孔边和肋端结果的可信度。”

#### 精度模式

\[
\min_{\mathbf l}\ N(\mathbf l)
\quad
\text{s.t.}\quad
E(\mathbf l)\le\varepsilon.
\]

用户语言：

> “目标量变化控制在 3% 内，尽量少用单元。”

如果需要给用户多种选择，应输出 Pareto 前沿，而不是要求普通用户自己设置多个抽象权重。

## 6. PSO 的复杂度应该怎样研究

### 6.1 标准 PSO 的真实成本

设：

- \(P\)：粒子数量；
- \(G\)：迭代代数；
- \(m\)：候选区域维度；
- \(C_{FE}(\mathbf l)\)：一次有限元评估成本。

粒子位置与速度更新的代价约为：

\[
O(GPm).
\]

但在有限元网格优化中，真正主导成本的是适应度评估：

\[
T_{\text{total}}
\approx
\sum_{k=1}^{N_{FE}} C_{FE}(\mathbf l_k)
+
O(GPm).
\]

标准实现通常有：

\[
N_{FE}\le GP.
\]

因此研究重点不应只写“把 PSO 从某个大 O 改到另一个大 O”，而应测量：

- 真实 FE 调用次数；
- 高保真 FE 调用次数；
- 总墙钟时间；
- 并行利用率；
- 缓存命中率；
- 达到同等质量所需评估数。

除非把 \(P\) 和 \(m\)视为常数，否则不能诚实地宣称一般 PSO 已变成 \(O(G)\)。即便算法更新近似为 \(O(G)\)，昂贵的有限元成本仍然存在。

### 6.2 建议的改进算法：MeshPSO

暂定名称：

> **MeshPSO：热点降维、缓存、多保真与代理辅助的离散粒子群网格优化。**

它不以发明复杂速度公式为核心，而是减少真正昂贵的求解次数。

#### 机制一：热点降维

从全模型的几千或几万个单元变量，降为 \(m=4\sim8\) 个区域等级变量。

搜索空间从不可处理的逐单元组合变为：

\[
4^m.
\]

当 \(m=6\) 时为 4096 种，当 \(m=8\) 时为 65536 种。PSO才有实际意义。

#### 机制二：离散更新与约束修复

粒子内部可保留连续速度，但位置映射到 L0–L3。每次更新后执行 repair：

- 满足强制区域下限；
- 满足局部上限；
- 若预计单元数超预算，优先降低边际收益低的区域；
- 消除不合法网格组合。

#### 机制三：确定性缓存

对粒子编码、几何版本、材料、载荷和模板版本计算哈希。完全相同的组合直接读取历史结果。

若缓存命中率为 \(h\)，真实评估数近似为：

\[
N_{FE}^{\text{real}}
=(1-h)N_{FE}^{\text{requested}}.
\]

#### 机制四：热启动

初始粒子不全随机，而是包含：

- 基础网格；
- 热点直接分配；
- 用户强制区域方案；
- 上一轮交互的最优解；
- 上一项目的相似模板解。

这直接服务于多轮智能体交互。

#### 机制五：自适应粒子规模

初期保持一定探索，后期根据种群多样性和停滞程度缩减粒子数。已有动态种群 PSO 研究表明，粒子数量可以随进化状态调整。[R12][R13]

建议简单实现：

- 初始 \(P_0=16\sim24\)；
- 达到稳定区域后缩减至 \(P_{min}=6\sim8\)；
- 长时间停滞时只增加少量探索粒子，而不是恢复整个大种群。

#### 机制六：多保真评估

三级评估：

1. **F0：解析/经验估计**——预测单元数、网格可行性、资源约束；
2. **F1：粗精度求解**——筛掉明显较差粒子；
3. **F2：高保真求解**——只评估精英和高不确定性候选。

#### 机制七：在线代理模型

有限元属于昂贵黑箱，代理辅助 PSO 已被用于减少真实评估次数。[R14][R15][R16]

对于本项目，代理模型输入为离散等级向量 \(\mathbf l\)，输出为：

- 目标误差或目标量变化；
- 单元数；
- 失败概率。

每代仅选择：

- 预测最优的前 \(k\) 个粒子；
- 预测不确定性最大的少量粒子；
- 必要的随机探索粒子；

进行真实 FE。

真实评估可由：

\[
GP
\]

降低为近似：

\[
N_{FE}=N_{init}+Gk,
\qquad k\ll P.
\]

#### 机制八：异步并行

每个真实粒子求解彼此独立，应采用异步队列：

- 先完成的粒子立即更新缓存和代理；
- 不等待最慢粒子形成同步屏障；
- 失败粒子自动重试一次，仍失败则记录为不可行。

### 6.3 改进 PSO 必须与简单方法比较

PSO不是天然最优。应比较：

- 直接热点分配；
- 贪心边际加密；
- 完全枚举（当 \(m\le5\) 时）；
- 标准离散 PSO；
- MeshPSO；
- 贝叶斯优化或 TPE；
- 随机搜索。

若 MeshPSO 只比标准 PSO 快，但仍比贪心或贝叶斯优化差，则不能把 PSO 作为核心贡献。

## 7. 业务闭环

### 7.1 组织角色

| 角色 | 责任 |
|---|---|
| 模板专家 | 建立模型、材料、边界、验证算例和适用范围 |
| CAE 管理员 | 发布模板、管理版本、资源和权限 |
| 相邻专业用户 | 提供工程意图、参数、重点区域并判断业务场景 |
| MeshPilot 智能体 | 编译任务、编排工具、优化、验证、记录和升级 |
| 审批专家 | 对 R2/R3 结果复核并承担最终工程判断 |
| 项目负责人 | 选择方案、管理时间和成本，不修改数值核心 |

### 7.2 一次完整用户旅程

用户输入：

> “这是钢箱梁横隔板局部模型。材料 Q345，左侧约束，右侧施加竖向分布荷载。孔边和第二根加劲肋端部是重点，总单元不超过八万，给我精度和单元数折中的三个方案。”

智能体执行：

1. 识别模板：钢箱梁横隔板局部模型；
2. 解析材料、载荷、重点区域和预算；
3. 检查缺失的板厚、荷载大小、单位和约束细节；
4. 展示模型卡，用户确认；
5. 生成基础网格并求解；
6. 合并用户区域和自动热点，形成 4–8 个候选区；
7. MeshPSO 搜索 L0–L3 分配；
8. 对候选 Pareto 解做高保真验证；
9. 输出三套方案、误差证据、资源和限制；
10. 用户说“孔边少一点、肋端再细一级”；
11. 智能体修改约束并热启动下一轮；
12. R2 任务提交专家复核。

### 7.3 购买者、使用者和受益者

- **购买/推动者**：设计院数字化负责人、CAE 负责人、工程软件负责人、制造企业研发管理者；
- **直接使用者**：结构设计、细节设计、BIM、工艺、检测养护和初级分析人员；
- **审核者**：高级 CAE/结构专家；
- **受益者**：项目团队、客户、制造和养护部门。

### 7.4 最有价值的业务场景

1. **高频重复变型**：同一构件模板反复改变尺寸、孔、肋、材料或载荷；
2. **专家资源稀缺**：设计团队等待 CAE 部门反馈；
3. **局部模型标准化程度高**：横隔板、节点、支座局部、连接板等；
4. **早期筛选价值高**：越早发现风险，返工代价越低；
5. **需要审计与知识沉淀**：大型组织、长期项目和多团队协作；
6. **用户有明确业务知识但不具备网格专长**。

### 7.5 不适合的业务场景

- 单次、原创、极高风险的研究模型；
- 输入条件本身高度不确定且无法确认；
- 缺少可验证模板；
- 结果不经过任何人工复核就直接用于安全签字；
- 模型变化大到每次都需要重新开发本构和求解流程。

## 8. 建议的首个三维 MVP

### 8.1 构件模板

首个模板建议采用：

> **三维钢箱梁横隔板局部构件，开孔可选，带参数化加劲肋。**

可配置：

第一阶段采用线弹性，核心研究是人机交互、网格优化和业务流程，不需要用复杂非线性人为增加难度。

### 8.2 输出

- 最大位移；
- 指定区域应力统计；
- 单元数和墙钟时间；
- 网格质量；
- L0–L3 区域分配；
- Pareto 方案；
- 分析合同与可追溯报告；
- “可用于早期筛选/需专家复核”的风险标签。

### RQ4：质量和治理

验证门、适用域检查和专家升级机制能否降低非专家误用风险，并提供足够的可追溯性？

## 10. 实验设计

### 10.1 数值实验

比较：

- 均匀网格；
- 热点直接加密；
- 热点 + 标准 PSO；
- 热点 + MeshPSO；
- 用户指定热点 + MeshPSO；
- 贪心、贝叶斯优化和小规模枚举。

指标：

- 目标量误差；
- 单元数；
- 高保真 FE 次数；
- 总 FE 次数；
- 墙钟时间；
- 缓存命中率；
- 代理筛选命中率；
- 约束违反率；
- Pareto hypervolume 或目标覆盖率。

### 10.2 用户研究

不招募随机公众，建议三组：

1. CAE/有限元专家；
2. 相邻专业人员：结构设计、BIM、细节设计、工艺、检测养护；
3. 具有工程背景的初级工程师或研究生。

任务：

- 从自然语言建立一个标准局部模型；
- 修改重点区域和预算；
- 在三个 Pareto 方案中选取一个；
- 发现系统故意设置的缺失材料或单位冲突；
- 判断何时需要专家升级。

指标：

- 有效任务完成率；
- 首次建立有效模型的时间；
- 用户交互次数；
- 输入错误率；
- 对模型假设的理解程度；
- 对自动化结果的校准信任；
- 专家复核修改量；
- 使用传统工具与智能体的时间差。

### 10.3 业务指标

- 单个模板的开发成本；
- 模板复用次数；
- 每次分析节省的专家时间；
- 从设计请求到结果的周转时间；
- 专家升级比例；
- 重复建模减少量；
- 可追溯报告覆盖率；
- 模板维护成本。

## 11. 系统边界与验收标准

### 11.1 智能体不得做的事

- 任意生成或修改求解器核心；
- 自动编造缺失的关键工程参数；
- 绕过网格质量、平衡或收敛检查；
- 把热点图等同于严格误差证明；
- 把早期筛选结果包装成最终安全验算；
- 在超出模板适用域时继续给出确定结论。

### 11.2 MVP 验收标准

- 支持至少一个三维桥梁局部构件模板；
- 支持自然语言分析合同；
- 支持用户指定区域和 L0–L3 约束；
- 支持热点候选生成；
- 支持标准 PSO 与 MeshPSO；
- 支持 Abaqus 或 CalculiX 中至少一个真实求解后端；
- 输出 2–3 个可解释 Pareto 方案；
- 所有运行具有可追溯记录；
- 能识别缺失参数、单位冲突和超模板任务；
- R2/R3 任务明确要求专家复核。

## 12. 推荐论文定位

中文暂定：

> **面向相邻专业工程人员的自然语言三维有限元局部网格优化智能体：热点引导与代理辅助离散粒子群方法**

英文暂定：

> **MeshPilot: A Natural-Language Agent for User-Guided 3D Finite-Element Mesh Optimization with Hotspot-Reduced Surrogate-Assisted Discrete PSO**

论文主张应保持克制：

- 不是让所有人都成为有限元专家；
- 不是让 LLM 替代数值求解器；
- 不是声称热点等于严格误差；
- 而是把专家验证过的三维局部分析和网格优化流程，可靠地交付给具有相关工程知识的更大用户群体。
